= {product} support

[role="_abstract"]
The {company-name} Customer Portal provides resources to help you troubleshoot issues and get support for {product}.
You can use the {company-name} Customer Portal for the following purposes:

* To search or browse through the Red Hat Knowledgebase of technical support articles about Red Hat products.
* To create a support case for {company-name} Global Support Services (GSS), select *{product}* as the product and select the appropriate product version.

== Collect diagnostic data for support cases

When opening a support case for {product-very-short} deployment, configuration, or runtime issues, providing comprehensive diagnostic data accelerates troubleshooting and reduces time to resolution.

The {product-very-short} must-gather tool in {product} 1.10.0 and later, collects diagnostic data from {product-very-short} deployments in a single operation.

This tool gathers platform information, deployment configurations, application logs, routes or ingress configurations, and namespace state in a standardized format optimized for support analysis.

The {product-very-short} must-gather tool is available on {ocp-brand-name} ({ocp-short}) {ocp-version-min} and later using the `oc adm must-gather` command, and on Kubernetes platforms including {aks-brand-name}, {eks-brand-name}, and {gke-brand-name} using a Helm chart deployment.

When preparing a support case:

. Collect diagnostic data using the {product-very-short} must-gather tool for your platform.
. Archive the collected output directory: `tar -czf must-gather-output.tar.gz must-gather.local.<timestamp>` (on {ocp-short}) or `tar -czf must-gather-output.tar.gz must-gather-output` (on Kubernetes).
. Attach the archive to your support case when creating or updating the case.
. Reference the must-gather output in your case description to help support engineers locate the diagnostic data.

For detailed must-gather collection procedures, configuration options, and platform-specific guidance, see {diagnostic-data-collection-book-link}[{diagnostic-data-collection-book-title}].
